Relasi (Relationship)

Diposting pada 04 Oktober 2025 | Oleh Cynthia Juniartien | Kategori: Relasi

Pengertian Relasi

Relasi adalah hubungan antar entitas yang menunjukkan bagaimana satu entitas berinteraksi dengan entitas lain. Relasi menjelaskan keterkaitan data dalam sistem.

Contoh Umum

  • Mahasiswa mengambil Mata Kuliah
  • Pelanggan melakukan Transaksi
  • Dosen mengajar Kelas

Kata seperti mengambil, melakukan, mengajar → menunjukkan relasi antar entitas.

Contoh Kehidupan Sehari-hari

Di perpustakaan:

Anggota meminjam Buku

Artinya, ada hubungan antara entitas Anggota dan Buku melalui relasi meminjam. Biasanya relasi ini juga bisa punya atribut sendiri, misalnya:

  • Tanggal Peminjaman
  • Tanggal Pengembalian
  • Status Peminjaman

Kardinalitas Relasi

Kardinalitas menunjukkan jumlah instance entitas yang dapat berpartisipasi dalam relasi:

  • One-to-One (1:1): Satu entitas A berhubungan dengan satu entitas B
  • One-to-Many (1:N): Satu entitas A berhubungan dengan banyak entitas B
  • Many-to-Many (M:N): Banyak entitas A berhubungan dengan banyak entitas B

Contoh Implementasi dalam SQL:

-- One-to-Many: Satu jurusan memiliki banyak mahasiswa
CREATE TABLE Jurusan (
  ID INT PRIMARY KEY,
  Nama_Jurusan VARCHAR(50)
);

CREATE TABLE Mahasiswa (
  NIM VARCHAR(10) PRIMARY KEY,
  Nama VARCHAR(100),
  Jurusan_ID INT,
  FOREIGN KEY (Jurusan_ID) REFERENCES Jurusan(ID)
);

-- Many-to-Many: Banyak mahasiswa mengambil banyak mata kuliah
CREATE TABLE Mahasiswa_MataKuliah (
  NIM VARCHAR(10),
  Kode_MK VARCHAR(10),
  Semester VARCHAR(10),
  PRIMARY KEY (NIM, Kode_MK),
  FOREIGN KEY (NIM) REFERENCES Mahasiswa(NIM),
  FOREIGN KEY (Kode_MK) REFERENCES Mata_Kuliah(Kode)
);
