Penjelasan Kunci pada Basis Data (Database Keys)

Diposting pada 08 Oktober 2025 | Oleh Cynthia Juniartien | Kategori: Database Keys

Dalam perancangan ERD (Entity Relationship Diagram), terdapat tiga jenis kunci utama yang berfungsi menjaga integritas serta keterhubungan data antar tabel: Primary Key, Foreign Key, dan Candidate Key. Ketiganya memiliki peran berbeda namun saling mendukung dalam pengelolaan data.

ğŸ”‘ 1. Primary Key (Kunci Utama)

Pengertian:

Primary Key adalah atribut (kolom) unik yang digunakan untuk mengenali setiap baris data secara spesifik dalam suatu tabel. Setiap tabel hanya memiliki satu Primary Key, dan nilainya tidak boleh kosong (NULL) maupun berulang (duplicate).

Fungsi:

  • Mengidentifikasi data secara unik.
  • Menjadi acuan utama hubungan antar tabel melalui Foreign Key.

Contoh Penerapan dalam Studi Kasus:

Nama Tabel Primary Key Keterangan
mahasiswa nim Nomor induk unik untuk setiap mahasiswa.
dosen id_dosen Identitas unik setiap dosen.
mata_kuliah kode_mk Kode unik untuk tiap mata kuliah.
kelas id_kelas Identitas unik untuk setiap kelas.
nilai id_nilai Nomor unik untuk setiap data nilai mahasiswa.

ğŸ” 2. Foreign Key (Kunci Tamu)

Pengertian:

Foreign Key adalah atribut yang digunakan untuk menghubungkan satu tabel dengan tabel lain. Foreign Key mereferensikan Primary Key dari tabel lain, menciptakan relasi antar entitas agar data tetap konsisten.

Fungsi:

  • Menjaga hubungan antar tabel.
  • Menghindari ketidaksesuaian atau kehilangan data antar entitas yang saling berkaitan.

Contoh Penerapan dalam Studi Kasus:

Tabel yang Memiliki FK Foreign Key Mengacu ke Arti Hubungan
kelas kode_mk mata_kuliah(kode_mk) Setiap kelas terkait dengan satu mata kuliah.
kelas id_dosen dosen(id_dosen) Setiap kelas diajar oleh satu dosen.
nilai nim mahasiswa(nim) Setiap nilai dimiliki oleh satu mahasiswa.
nilai id_kelas kelas(id_kelas) Setiap nilai berasal dari satu kelas tertentu.
nilai kode_mk mata_kuliah(kode_mk) Menunjukkan nilai dari mata kuliah tertentu.

ğŸ§  3. Candidate Key (Kunci Kandidat)

Pengertian:

Candidate Key adalah atribut unik lain dalam tabel yang berpotensi menjadi Primary Key tetapi tidak dipilih sebagai Primary Key utama. Semua Candidate Key juga harus memiliki nilai unik dan tidak boleh kosong.

Fungsi:

  • Menjadi alternatif pengenal unik selain Primary Key.
  • Menjamin tidak adanya duplikasi data penting seperti email, kode, atau nomor identitas.

Contoh Penerapan dalam Studi Kasus:

Tabel Candidate Key Keterangan
mahasiswa email Email mahasiswa bersifat unik.
dosen email_dosen, no_induk_dosen Dosen memiliki dua identitas unik selain ID utama.
mata_kuliah kode_unik Setiap mata kuliah memiliki kode unik tambahan.
kelas kode_kelas Kode kelas digunakan untuk identifikasi cepat (misal: KLS2025A).
nilai kode_transaksi Kode transaksi unik untuk setiap catatan nilai.

ğŸ§± Kesimpulan

Jenis Kunci Jumlah Fungsi Utama
Primary Key 5 Mengidentifikasi setiap data secara unik di masing-masing tabel.
Foreign Key 5 Menghubungkan data antar tabel agar saling terintegrasi.
Candidate Key 5 Menjadi alternatif pengenal unik selain Primary Key.
"Mahasiswa dengan nim = 23011001 terdaftar di kelas dengan id_kelas = 10, kelas tersebut diajar oleh dosen dengan id_dosen = 5, dan berkaitan dengan mata kuliah kode_mk = MK101, nilai akhirnya tercatat pada tabel nilai dengan kode_transaksi = TRX1010A."

Semua data saling terhubung otomatis melalui Primary Key dan Foreign Key, sementara Candidate Key memastikan tidak ada data ganda seperti email, kode kelas, atau transaksi.
