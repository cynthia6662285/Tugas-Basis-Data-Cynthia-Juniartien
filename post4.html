Mengenal Physical Model

Diposting pada 27 September 2025 | Oleh Cynthia Juniartien | Kategori: Physical Model

Pengertian Physical Model

Physical Model adalah model basis data yang menggambarkan bagaimana data benar-benar disimpan di dalam komputer. Berbeda dengan conceptual model yang hanya menjelaskan hubungan antar data secara umum, physical model sudah masuk ke detail teknis, seperti bentuk tabel, nama kolom, tipe data yang digunakan, serta aturan seperti primary key, foreign key, atau index.

Implementasi Nyata

Dengan kata lain, physical model merupakan rancangan nyata yang nantinya bisa langsung diterjemahkan ke dalam perintah SQL untuk membangun basis data di sistem manajemen basis data.

Contoh Physical Model:

-- Tabel Mahasiswa
CREATE TABLE Mahasiswa (
  NIM INT PRIMARY KEY,
  Nama VARCHAR(100) NOT NULL,
  Tanggal_Lahir DATE,
  Alamat TEXT,
  Jurusan_ID INT,
  FOREIGN KEY (Jurusan_ID) REFERENCES Jurusan(ID)
);

-- Tabel Jurusan
CREATE TABLE Jurusan (
  ID INT PRIMARY KEY AUTO_INCREMENT,
  Nama_Jurusan VARCHAR(50) UNIQUE NOT NULL,
  Fakultas VARCHAR(50)
);

Perbedaan Conceptual vs Physical Model

Aspect Conceptual Model Physical Model
Tujuan Memahami kebutuhan bisnis Implementasi teknis
Detail High-level, umum Detail teknis lengkap
Pemirsa Pengguna bisnis Developer & DBA
Output ER Diagram SQL Script

Komponen Physical Model

  • Tabel: Struktur penyimpanan data
  • Kolom: Field dalam tabel dengan tipe data spesifik
  • Primary Key: Identifier unik untuk setiap record
  • Foreign Key: Hubungan antar tabel
  • Index: Optimasi pencarian data
  • Constraint: Aturan integritas data

Dari gambaran physical model inilah database dapat dibuat dan dijalankan secara nyata di komputer.
