üß† Mengenal SQL, DDL, DML, DCL, dan TCL dalam Basis Data

Diposting pada 28 Oktober 2025 | Oleh Cynthia Juniartien | Kategori: SQL & Database

Dalam dunia basis data, istilah seperti SQL, DDL, DML, DCL, dan TCL sering muncul. Semua istilah ini berhubungan erat dengan bagaimana kita membuat, mengelola, dan memanipulasi data di dalam sistem database seperti MySQL, PostgreSQL, atau Oracle.

üíæ Apa Itu SQL?

SQL (Structured Query Language) adalah bahasa standar untuk mengelola dan memanipulasi basis data relasional. Dengan SQL, kita bisa:

  • Membuat dan menghapus tabel
  • Menambahkan, mengubah, atau menghapus data
  • Mengatur hak akses pengguna
  • Mengontrol transaksi agar data tetap konsisten

Sederhananya, SQL adalah bahasa komunikasi antara manusia dan database.

-- Contoh perintah SQL:
SELECT * FROM Pelanggan WHERE Kota = 'Karawang';

Perintah di atas digunakan untuk menampilkan semua data pelanggan yang berasal dari Karawang.

üß± 1. DDL (Data Definition Language)

DDL adalah bagian dari SQL yang digunakan untuk mendefinisikan struktur database seperti tabel, kolom, dan relasi.

üîπ Tujuan utama DDL adalah membentuk "kerangka" database.

Contoh perintah DDL:

Perintah Fungsi
CREATE Membuat tabel atau database baru
ALTER Mengubah struktur tabel (misalnya menambah kolom)
DROP Menghapus tabel atau database
TRUNCATE Menghapus semua isi tabel tanpa menghapus strukturnya
-- üìò Contoh:
CREATE TABLE Produk (
    ID_Produk INT PRIMARY KEY,
    Nama_Produk VARCHAR(100),
    Harga DECIMAL(10,2)
);

‚úçÔ∏è 2. DML (Data Manipulation Language)

DML berfungsi untuk memanipulasi data di dalam tabel, bukan strukturnya. Bagian ini paling sering digunakan dalam aktivitas sehari-hari seperti menambah, mengedit, dan menampilkan data.

Contoh perintah DML:

Perintah Fungsi
SELECT Menampilkan data dari tabel
INSERT Menambahkan data baru
UPDATE Mengubah data yang sudah ada
DELETE Menghapus data
-- üìò Contoh:
INSERT INTO Produk (ID_Produk, Nama_Produk, Harga)
VALUES (1, 'Pomade Super Hold', 45000);

üîê 3. DCL (Data Control Language)

DCL digunakan untuk mengatur hak akses dan keamanan data di dalam database. Bagian ini penting jika database digunakan oleh banyak pengguna dengan tingkat akses berbeda-beda.

Contoh perintah DCL:

Perintah Fungsi
GRANT Memberikan hak akses ke pengguna
REVOKE Menghapus hak akses dari pengguna
-- üìò Contoh:
GRANT SELECT, INSERT ON Produk TO user_daino;
REVOKE DELETE ON Produk FROM user_daino;

üîÑ 4. TCL (Transaction Control Language)

TCL berhubungan dengan pengelolaan transaksi dalam database ‚Äî memastikan data tetap konsisten ketika ada banyak perubahan sekaligus.

Transaksi biasanya mencakup lebih dari satu perintah DML, dan semua perintah itu harus berhasil semua atau batal semua.

Contoh perintah TCL:

Perintah Fungsi
COMMIT Menyimpan semua perubahan transaksi
ROLLBACK Membatalkan transaksi (mengembalikan data ke kondisi awal)
SAVEPOINT Menandai titik tertentu di dalam transaksi untuk rollback sebagian
-- üìò Contoh:
START TRANSACTION;

UPDATE Produk SET Harga = 40000 WHERE ID_Produk = 1;
DELETE FROM Pelanggan WHERE ID_Pelanggan = 5;

ROLLBACK; -- Membatalkan semua perubahan di atas

üß© Kesimpulan

Jenis SQL Kepanjangan Fungsi Utama Contoh Perintah
DDL Data Definition Language Mengatur struktur database CREATE, ALTER, DROP
DML Data Manipulation Language Mengolah data di dalam tabel SELECT, INSERT, UPDATE, DELETE
DCL Data Control Language Mengatur hak akses dan keamanan GRANT, REVOKE
TCL Transaction Control Language Mengelola transaksi database COMMIT, ROLLBACK, SAVEPOINT
"Memahami perbedaan antara DDL, DML, DCL, dan TCL membuat kita lebih mudah dalam merancang sistem database yang efisien, aman, dan stabil. SQL bukan hanya soal mengambil data, tetapi juga tentang bagaimana menjaga integritas, keamanan, dan performa dari data itu sendiri."

‚ú® Penutup

Dengan memahami berbagai komponen SQL ini, kita dapat lebih efektif dalam mengelola database dan memastikan data tetap aman, konsisten, dan terstruktur dengan baik.
